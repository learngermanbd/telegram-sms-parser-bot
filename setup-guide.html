🤖 Telegram SMS Parser Bot

Setup & workflow guide — automatically parses SMS messages (bKash, NAGAD, Rocket) and sends formatted templates to your private channel.

🧰 How It Works

This bot receives forwarded SMS messages, intelligently parses transaction details, and sends structured templates to your private channel. No keyword list needed.

🔄 Also saves to SQLite database for summaries & search 🔄

Supported providers:

bKash NAGAD Rocket / Nexus Pay

The bot automatically detects whether a message is a Cash Out or Money Received transaction and formats it accordingly. Unrecognized messages are forwarded as raw text with a generic prefix.

Security filter: OTPs, passwords, PINs, CVVs, 2FA codes, recovery codes, and similar sensitive content are silently blocked — they never reach the channel.

Transaction database: Every parsed transaction is saved to a local SQLite database (transactions.db). You can query it anytime with commands.

No .env file needed — all config lives directly in telegram_bot.py.

📋 Prerequisites

  • Python 3.10+ installed
  • A Telegram account
  • A private Telegram channel to forward messages to
  • An SMS forwarding app (e.g. sms-fw.com) on your phone
  • A Render account (free tier works) — sign up
  • Git & a GitHub / GitLab repo

1 Create a Telegram Bot

  1. Open Telegram and search for @BotFather.
  2. Send: /newbot
  3. Choose a name (e.g. SMS Parser Bot) and a username ending in bot.
  4. @BotFather replies with your HTTP API token. Save it — this is your BOT_TOKEN.

2 Get Your Chat ID

The bot listens in your private chat with it. Your SMS forwarder will send messages to this chat, and the bot will read them there.

  1. Open a chat with your new bot and send any message (e.g. "hi").
  2. Visit in your browser:
    https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates
  3. Find the "chat":{"id": ...} field. That number is your SOURCE_CHAT_ID.
💡 Tip: Your personal chat ID is usually a positive number like 123456789.

3 Get the Channel ID

  1. Add your bot as an administrator to your private channel.
  2. Post any message in the channel.

5 Configure SMS Forwarding (sms-fw.com)

You need an SMS forwarding app on your phone that sends SMS messages to your bot's private chat. sms-fw.com is a popular choice.

  1. Install SMS Forwarder — Download sms-fw.com on your Android phone (or use any SMS forwarding app that supports Telegram).
  2. Set up Telegram forwarding:
    • In the app, add a new forwarding rule.
    • Select Telegram as the target.
    • Enter your bot's username (e.g. @SMSparserBot).
    • Start a private chat with your bot first, then forward a test SMS.
  3. Filter by sender: Create rules for each provider:
    • Rule for bKash — forward messages from "bKash" or "16216"
    • Rule for NAGAD — forward messages from "NAGAD"
    • Rule for Rocket / Nexus Pay — forward messages from "Rocket" or related numbers
    • Optional: Add an "all SMS" rule (the bot filters OTPs/PINs automatically)
⚠️ Important: The SMS forwarder sends messages to the same private chat with the bot you tested in Step 2. Make sure SOURCE_CHAT_ID matches that chat.
✅ Tip: Send a test SMS to yourself and verify the formatted template appears in your channel before deploying.

7 Deploy to Render

The render.yaml Blueprint makes it one-click:

  1. Push to GitHub / GitLab 
    git add .
git commit -m "Add Telegram SMS parser bot"
git push
  2. Log into render.com.
  3. Click "New +" → "Blueprint".
  4. Connect your repo — Render auto-detects render.yaml. Click "Apply".
  5. Enter the secrets (BOT_TOKEN, SOURCE_CHAT_ID, DESTINATION_CHANNEL_ID) when prompted.
  6. Click "Deploy". Wait 2-3 minutes.
📡 Health check: Render pings /health every few seconds, keeping the bot awake 24/7.
⚠️ Note: The SQLite database (transactions.db) is ephemeral on Render — it resets on each deploy. For persistent storage, databases like Supabase or Railway are better suited.

✅ Your bot is live! SMS messages forwarded to your bot are automatically parsed and sent to your channel as formatted templates.

🎮 Commands Reference

Send these commands in your private chat with the bot (not the channel):

Command Description
/daily Shows today's transaction summary (count & total per provider)
/weekly Shows this week's transaction summary (last 7 days)
/search <TrxID> Search transactions by Trx ID (partial match)
/search amount <amount> Search transactions by amount (e.g. /search amount 60)
/start Shows a brief welcome message with available commands
💡 Search tip: After /search returns a numbered list, reply with the number (1, 2, ...) to see the full formatted template for that transaction.

🚧 Troubleshooting

  • Bot not responding: Make sure SOURCE_CHAT_ID matches your personal chat ID (not a group ID). Send any message to the bot and check /getUpdates.
  • Messages not reaching channel: Verify the bot is an admin of the destination channel (not just a member).
  • "FATAL: You must edit...": You forgot to replace a placeholder in telegram_bot.py. Search for "YOUR_" in the file and replace all matching strings.
  • Message forwarded but showing as raw text (not parsed): The SMS format may differ from the parser's expectations. Check the bot's logs — the parser prints which service it detected or why it failed. You can add new regex patterns to the parser if needed.
  • OTP / password not showing up in channel: This is working as intended! The security filter blocks sensitive messages. Check the bot logs for "🔒 Blocked: sensitive content" warnings.
  • Search returning no results: Only messages that were successfully parsed are saved to the database. Raw/unrecognized forwards are not stored.
  • Commands not working (/daily, /search): Send commands in your private chat with the bot, not in the channel.
  • Database empty on Render: Render's filesystem is ephemeral. On restart/deploy, transactions.db resets. Consider persistent storage (Supabase) for long-term history.
